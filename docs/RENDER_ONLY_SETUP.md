# Render Blueprint Deployment — Quick Setup

This guide covers the minimal steps to deploy the GUTMANN backend and frontend on Render using the `render.yaml` Blueprint.

---

## Prerequisites

- A [Render](https://render.com) account connected to GitHub
- A [Supabase](https://supabase.com) project with credentials ready (URL, anon key, service-role key)

---

## Steps

### 1. Fork / Push to GitHub

Ensure the repository is available in your GitHub account. No special branch configuration is needed.

### 2. Connect GitHub to Render

1. Log in to [Render](https://render.com).
2. Click **New → Blueprint**.
3. Select the repository containing this code (or your fork).
4. Render detects `render.yaml` automatically — click **Apply**.

### 3. Set Secret Environment Variables

Render will prompt you to fill in the `sync: false` variables during Blueprint setup. Provide:

| Variable | Where to find it |
|---|---|
| `SUPABASE_URL` | Supabase → Settings → API → Project URL |
| `SUPABASE_ANON_KEY` | Supabase → Settings → API → anon/public key |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase → Settings → API → service_role key (keep secret) |
| `REACT_APP_API_URL` | Your backend service URL + `/api` — set after services are created, e.g. `https://<your-backend-name>.onrender.com/api` |

> **Never commit real secret values to the repository.** Always set them in the Render dashboard.

`JWT_SECRET` is auto-generated by Render (`generateValue: true`) — no action needed.

`FRONTEND_URL` is automatically set to the frontend static site URL via Render's `fromService` reference.

### 4. Deploy

Click **Apply** / **Create Services**. Render will:

1. Build and start the **gutmann-backend** Node service (`cd backend && npm ci` then `cd backend && npm start`).
2. Build and publish the **gutmann-frontend** static site (`cd frontend && npm ci && npm run build`, published from `frontend/build`).

Wait for both services to show **Live** status (usually 5–15 minutes on first deploy).

### 5. Verify

- **Backend health:** `https://<your-backend-name>.onrender.com/api/health` → `{ "status": "ok" }`
- **Frontend:** `https://<your-frontend-name>.onrender.com` → Login page loads

---

## Redeployment

Push any commit to `main` and Render automatically redeploys both services.

To redeploy manually: Render Dashboard → your service → **Manual Deploy → Deploy latest commit**.
